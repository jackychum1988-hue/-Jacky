# 忍者指令 (Ninja Code) — 儿童编程启蒙益智游戏设计文档

> 日期：2026-06-19 | 状态：设计完成，待审核

## 1. 项目概述

为一款面向9岁男孩的网页端逻辑推理益智游戏。玩家扮演忍者学徒，通过排列指令序列控制忍者在道场网格中移动，避开障碍、到达目标，从而锻炼编程思维（序列、方向、循环）。

| 维度 | 选择 |
|------|------|
| 核心能力 | 逻辑推理 |
| 玩法类型 | 路径规划 / 编程启蒙（指令序列点击式） |
| 主题风格 | 🥷 忍者像素复古（8-bit像素风 + 霓虹绿配色） |
| 目标用户 | 9岁男孩 |
| 运行平台 | 电脑网页端（响应式，也兼容平板） |
| 关卡结构 | 预设关卡制（40关，4段位） |
| 技术栈 | React + Vite + TypeScript |
| 部署方式 | Vercel 静态部署 |
| 数据存储 | localStorage（无需登录） |

## 2. 核心玩法

### 2.1 游戏循环

```
选择关卡 → 观察地图 → 排列指令 → 执行观看 → 成功过关 / 失败重试
```

### 2.2 界面三区域布局

```
┌─────────────────────────────────┐
│  ① 地图区（道场网格）           │
│  🥷 忍者 · 🎯 卷轴 · 各种障碍   │
├─────────────────────────────────┤
│  ② 指令序列区（已添加的指令）    │
│  [↑前进] → [↪右转] → [↑前进]   │
├─────────────────────────────────┤
│  ③ 指令面板（按钮+执行）        │
│  [↑前进] [↩左转] [↪右转]       │
│  [🦘跳跃] [⚔️劈砍] [▶执行]     │
└─────────────────────────────────┘
```

### 2.3 指令集（5种）

| 指令 | 图标 | 颜色 | 作用 |
|------|------|------|------|
| 前进 | ↑ | 红色 #cc3333 | 面向方向前进一步 |
| 左转 | ↩ | 紫色 #7733cc | 原地左转90° |
| 右转 | ↪ | 紫色 #7733cc | 原地右转90° |
| 跳跃 | 🦘 | 橙色 #cc8833 | 向前跳过一格（越过坑洞） |
| 劈砍 | ⚔️ | 橙色 #cc8833 | 破坏面前一格的可破坏物 |

### 2.4 地图元素

| 元素 | 符号 | 说明 |
|------|------|------|
| 忍者 | 🥷 | 起点，有朝向属性（上下左右） |
| 卷轴 | 🎯 | 目标点，忍者到达即过关 |
| 墙壁 | 🧱 | 不可通过，不可破坏 |
| 木桩 | 🪵 | 可被⚔️劈砍破坏 |
| 坑洞 | ⛳ | 可通过🦘跳跃越过 |
| 勾玉 | 💎 | 可选收集品，集齐加分/3星 |

### 2.5 操作流程

1. 观察地图：忍者在哪？卷轴在哪？有什么障碍？
2. 点击指令按钮，按顺序添加指令
3. 点击已添加的指令可删除，支持全部清空
4. 点击 ▶ 执行，观看忍者逐条执行指令
5. 忍者到达卷轴 → 通关弹窗，评星
6. 撞墙/掉坑/超出边界 → 失败动画，可重试

## 3. 关卡设计

### 3.1 40关 · 4段位

| 段位 | 关卡 | 网格 | 解锁指令 | 核心挑战 |
|------|------|------|------|------|
| ⬜ 白带·入门 | 1-10 | 4×4 ~ 5×5 | ↑前进、↩左转、↪右转 | 方向感知、指令顺序 |
| 🟨 黄带·进阶 | 11-20 | 5×5 ~ 6×6 | + ⚔️劈砍 | 破坏障碍、多路线选择 |
| 🟩 绿带·挑战 | 21-30 | 6×6 ~ 7×7 | + 🦘跳跃 | 多种障碍组合、最优路径 |
| ⬛ 黑带·大师 | 31-40 | 7×7 ~ 8×8 | + 🔄循环(函数) | 复杂迷宫、用函数简化重复 |

### 3.2 难度曲线规则

- 每关必须在设计时计算「最优步数」（手动验证或算法计算）
- 前5关保证3步以内可解，确保孩子快速建立信心
- 每个段位最后1关是「Boss关」，难度跳升
- 黑带段位引入「函数」：可将一段指令序列封装为一个招式，重复调用

### 3.3 评星规则

| 评级 | 条件 |
|------|------|
| ⭐⭐⭐ 3星 | 实际步数 ≤ 最优步数（完美解） |
| ⭐⭐ 2星 | 实际步数 ≤ 最优步数 + 3 |
| ⭐ 1星 | 完成过关（步数超出但完成） |

## 4. 进度系统

### 4.1 存储

- 使用浏览器 `localStorage` 存储进度
- 无后端、无账号，刷新/关闭浏览器不丢失
- 数据结构：

```typescript
interface Progress {
  completedLevels: Record<number, { stars: number; bestSteps: number }>;
  totalStars: number;
  unlockedBelt: Belt; // 'white' | 'yellow' | 'green' | 'black'
}
```

### 4.2 解锁机制

- 关卡：N号关通关后解锁 N+1 号关
- 段位：白带→黄带需20星，黄带→绿带需40星，绿带→黑带需60星
- 已通关卡可随时重玩刷星

### 4.3 成就系统（彩蛋）

| 成就 | 触发条件 | 效果 |
|------|------|------|
| 🥷 完美忍者 | 单关3星 | 通关弹窗显示特殊像素动画 |
| ⚡ 闪电忍者 | 一次过（无重试） | 闪电粒子特效 |
| 💎 宝藏猎人 | 收集全部勾玉 | 隐藏像素角色出场 |

## 5. 视觉风格

### 5.1 像素复古（8-bit Pixel Retro）

- **背景色**：深蓝黑 `#0a0a2e`
- **网格底色**：暗蓝 `#1a1a4e`
- **网格边框**：蓝紫 `#3333aa`
- **主强调色**：霓虹绿 `#33ff33`（通关、成功、高亮）
- **行动指令**：红色系 `#cc3333`
- **转向指令**：紫色系 `#7733cc`
- **特殊指令**：橙色系 `#cc8833`
- **执行按钮**：亮绿 `#33cc33`
- **文字色**：白 `#ffffff` / 灰 `#aaa`
- **星星/金币**：金黄 `#ffd700`

### 5.2 字体

- 主字体：`'Courier New', 'Source Code Pro', monospace`（像素/等宽感）
- 标题用 font-weight 900，正文用 normal

### 5.3 UI元素风格

- 所有边框用2px实线 + 亮色边框（像素感）
- 按钮无圆角（border-radius: 0~2px）
- 文字使用 `text-shadow` 模拟像素阴影
- 过渡动画使用简单的闪烁/脉冲，无平滑过渡（保持8-bit手感）
- 关卡格子用方形块，已通过绿色边框、未解锁半透明

### 5.4 动画语言

- 忍者移动：逐格跳移（非平滑滑动），配合像素位移音效感
- 劈砍：目标格子闪烁2次后消失
- 跳跃：忍者Y轴抛物跳跃动画
- 通关：屏幕闪烁绿框 + 星星逐个弹出
- 失败：屏幕短暂红色闪烁 + 忍者回到起点

### 6.3 核心数据流

```
用户点击指令按钮
  → useGameState 添加指令到 sequence[]
  → SequenceBar 重渲染显示新序列
用户点击执行
  → engine.execute(level, sequence) 逐帧运行动画
  → 每500ms执行一条指令，更新忍者位置/朝向
  → 执行完毕：到达目标→弹出CompleteModal / 失败→重置
  → validator.calculateStars(steps, optimalSteps) → 保存到localStorage
```

### 6.4 关卡数据结构

```typescript
interface LevelData {
  id: number;
  name: string;           // "穿越木桩阵"
  belt: Belt;             // 'white' | 'yellow' | 'green' | 'black'
  gridSize: [number, number]; // [cols, rows]
  start: { x: number; y: number; facing: Direction };
  target: { x: number; y: number };
  walls: [number, number][];       // 墙壁坐标
  barriers: [number, number][];    // 木桩（可破坏）
  gaps: [number, number][];        // 坑洞（需跳跃）
  gems?: [number, number][];       // 勾玉（可选收集品）
  optimalSteps: number;            // 最优步数
  availableCommands: Command[];    // 本关可用指令
}
```

### 6.5 游戏状态机

```
IDLE (观察地图)
  → BUILDING (排列指令中)
  → EXECUTING (逐条执行指令动画)
  → SUCCESS (到达目标) / FAILED (撞墙/掉坑/越界)
  → IDLE (重试/下一关)
```

## 7. 边界条件 & 错误处理

- **网格边界**：前进超出网格 → 失败，忍者不动，提示"撞到道场墙壁了！"
- **撞墙**：前进到墙壁格子 → 失败，同上
- **坑洞未跳跃**：前进到坑洞且上一条不是跳跃 → 失败，"掉进陷阱了！试试跳跃指令"
- **木桩未劈砍**：前进到木桩 → 失败，"木桩挡住了路！先用劈砍破坏它"
- **指令上限**：每关最多8条指令（入门关3-5条），超出禁用添加按钮
- **空序列执行**：点击执行但无指令 → 按钮不响应，提示"先添加指令吧！"
- **localStorage不可用**：静默降级，游戏仍可玩但进度不保存（提示用户）
- **关卡数据完整性**：每个关卡定义加载时校验必填字段，缺字段显示错误边界

## 8. 非功能需求

- **首屏加载**：< 2秒（Vite打包后 < 100KB）
- **浏览器兼容**：Chrome/Firefox/Edge 最新两个大版本
- **响应式**：桌面优先，平板可玩（触屏可点击按钮）
- **无障碍**：按钮有 aria-label，支持键盘Tab导航
- **无外部依赖**：不调用外部API，纯本地运行
